Income Prediction Application

Help

Form Input

  1. Form input takes inputs from user, passes it to the prediction API and displays the prediction.
  2. If a user is not willing to disclose any of the requested information. He/she is free to leave it untouched. Such inputs are considered as missing values.
  3. The result is displayed on the upper portion of the page, below the banner. Errors, if any, will be displayed at the bottom of the page below the predict button.
 

DB Input

  1. This service assumes the data with independent features and the target are stored in a database. The target has some missing values which must be updated with the predictions.
  2. The user enters the DB credentials, the id/primary key column that uniquely identifies a data sample/row and the target column.
  3. The prediction of each sample/record is updated in the target field of the database using the id/primary key.
 

File Input

  1. This service assumes the data with independent features are stored in a CSV/Excel file and the prediction must be appended to each row/sample.
  2. The user uploads a CSV/Excel file with independent features and the target column name.
  3. The prediction of each sample/row is appended to each row/sample. Once all the rows/samples are appended with the prediction. The file is downloaded automatically in the same format as that of the uploaded file. Upon any error, the error is displayed to the user.
